Step 1 — DNS Setup
📶Router — All Devices at Once Recommended

Set DNS on your router once — every device is protected automatically with no per-device config.

1
Log in to your router admin panel (usually 192.168.1.1 or 192.168.0.1)
2
Find LAN Settings → DHCP Server
3
Set Primary DNS to {{ host_ip }}
4
Set Secondary DNS to 1.1.1.1 (fallback)
5
Save and reconnect devices
💻Per-Device Manual Setup

If you can't change the router, set DNS on each device individually:

  • Windows: Settings → Network & Internet → your connection → DNS server assignment → Manual → set IPv4 Preferred DNS to {{ host_ip }}
  • macOS: System Settings → Network → your connection → Details → DNS tab → add {{ host_ip }}
  • iPhone / iPad: Settings → Wi-Fi → tap ⓘ → Configure DNS → Manual → Add Server → {{ host_ip }}
  • Android: Settings → Wi-Fi → modify network → Advanced → IP Settings: Static → DNS 1: {{ host_ip }}
  • Smart TV / Console: Network Settings → IP Settings → Manual → DNS: {{ host_ip }}
Step 2 — CA Certificate (YouTube Ad Blocking)
Enables transparent YouTube ad blocking — pre-roll, mid-roll, and post-roll ads stripped server-side. Requires trusting the local CA certificate on each device. Skip if you only need DNS-level blocking.
📲iPhone / iPad
📥 Install Certificate Profile
1
Tap the button above in Safari — tap Allow to download
2
Settings → Profile Downloaded → Install → enter passcode → Install
3
Settings → General → About → Certificate Trust Settings → enable RichSinkhole CA
🍎macOS
📥 Install Certificate Profile
1
Click the button above — System Settings will open automatically
2
Privacy & Security → Profiles → Install
3
Open Keychain Access → find RichSinkhole CA → double-click → Trust → set to Always Trust
🖥Windows
📥 Download CA Certificate
1
Download and double-click richsinkhole-ca.crt → Install Certificate → Local Machine → Next
2
Select Trusted Root Certification Authorities → OK → Next → Finish
🐧Linux
curl -fsSL http://{{ host_ip }}/install-cert.sh | bash

Installs system-wide and injects into Firefox and Chrome/Chromium. Requires libnss3-tools (sudo apt install libnss3-tools). Restart your browser after.

📱Android
Android 7+ does not allow apps (including the YouTube app) to trust user-installed CA certificates, so YouTube in-video ad blocking is not possible on Android. DNS-level blocking (Step 1) still works for all other apps. For YouTube ads on Android, use YouTube ReVanced.
VERIFY IT'S WORKING
DNS blocking
Visit doubleclick.net — it should fail to load. Check the live log to see your device's queries.
YouTube proxy
Play a video — pre-roll and mid-roll ads should be skipped. Hard-refresh with Ctrl+Shift+R first.
